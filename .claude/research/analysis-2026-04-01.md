# Codebase Analysis — 2026-04-01

Three-agent audit covering code quality, documentation consistency, and monolith regression comparison.

## Critical (must fix)

- [ ] **C1** `x_assert` crashes when `exec_log` is None — missing null guard on `log_user_msg()` call at `metacommands/control.py:65`. All four other `exec_log` calls in `control.py` (lines 160, 172, 232, 283) have same issue. Pattern: `if _state.exec_log is not None:`.

- [ ] **C2** `_ping_db` version-query loop has `break` at wrong indentation (`cli/run.py:157-159`). Exits after first non-raising query even if no version returned. Fix: indent `break` inside the `if row and row[0]:` block.

- [ ] **C3** `gui_console_width()`/`gui_console_height()` refactored from setters (accepting width/height arg) to zero-argument getters (`utils/gui.py:361-368`), but callers in `system.py:139,145` still pass arguments. `CONSOLE SET WIDTH/HEIGHT` crashes with TypeError at runtime.

## High (should fix)

- [ ] **H1** `$ERROR_MESSAGE` contains short `str(e)` for non-halting errors (`engine.py:309,354`). Monolith used full `errmsg()` with location/timestamp. Fix: call `stamp_errinfo(e)` then use `e.errmsg()`.

- [ ] **H2** `log_status_exception` never called anywhere in refactored code (`fileio.py:533` defines it, zero call sites). Non-halting errors produce no log entry. Monolith auto-logged every error in `ErrInfo.__init__`.

- [ ] **H3** `project_context.md` says version 2.5.0; actual is 2.11.0. Completed table missing v2.6–v2.11.

- [ ] **H4** README options table missing `--ping`, `--profile`, `--output-dir`.

## Medium

- [ ] **M1** Module docstrings in `conditions.py` and `control.py` placed after imports — `__doc__` is None, breaks mkdocstrings/help().

- [ ] **M2** Dead `endloop()` function in `control.py:137-142` (never called; `state.endloop()` is authoritative).

- [ ] **M3** YAML `append=True` produces invalid YAML without `---` document separator. Fix: emit `---\n` before appended dump.

- [ ] **M4** `x_debug_log_subvars` log format wrong (`debug.py:77`) — prints full tuple instead of name/value. Fix: `f"Substitution [{s[0]}] = [{s[1]}]"`.

- [ ] **M5** Divergence doc says FEATHER uses "polars + pyarrow" — only polars is used (`divergence.md:37`).

- [ ] **M6** README pre-commit hook rev is `v2.4.4`, should be current version.

## Low / Suggestions

- [ ] **L1** `__delattr__` in `state.py:339` allocates fresh `RuntimeContext()` on every mock-patch restore. Fix: module-level `_DEFAULT_CTX` singleton.

- [ ] **L2** Dot-command parsing inconsistency in REPL — `_handle_dot_command` uses `line[1:].strip().lower()` but exit-check uses `line.lower().lstrip(".")`.

- [ ] **L3** `write_query_to_xlsx` single-sheet doesn't update `export_metadata` (multi-sheet variant does).

- [ ] **L4** `_ping_db` parameter missing type annotation.

- [ ] **L5** `lint.py:197` both branches of conditional expression are identical — redundant.

- [ ] **L6** `type()` equality instead of `isinstance()` in `engine.py:197`.
